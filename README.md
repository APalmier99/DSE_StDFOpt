# Stochastic Derivative-Free Optimization Benchmarks

This repository contains a clean, reproducible MATLAB implementation of several stochastic derivative-free optimization (DFO) algorithms and a unified experimental harness to compare them on a set of standard problems.

Algorithms included:
- DSE-DD (stochastic line-search variant)
- SDS (stochastic direct search)
- StoMADS (wrapped third-party implementation, lightly adapted to match our evaluation protocol)
- Random nonconvex search (Nesterov's baseline; see [Nesterov & Spokoiny, 2017](https://link.springer.com/article/10.1007/s10208-015-9296-2))
- Kornowski-Shamir noisy zeroth-order method (Algorithm 1 + Algorithm 2 implementation)

Key features:
- Sample-average noisy oracle with sample-size scaling m(h) ∝ h^{-4} for DSE-DD, SDS, and StoMADS
- Consistent logging of f(x_k) across algorithms
- Batch runner to iterate over multiple problems and multiple randomized runs

## Folder structure

- `experiments/` — drivers to run algorithms on a single problem or on a batch of problems
- `problems/` — problem definitions, each as a MATLAB function `y = problem_name(x)`
- `src/`
  - `algorithms/` — implementation of DSE-DD, SDS, StoMADS wrapper, random-search baseline, and Kornowski-Shamir baseline
  - `oracles/` — unified noise model and sample-exponent helper
- `third_party/StoMADS/` — StoMADS core code (kept separate). See their license for terms.
- `results/` — output `.mat` files generated by experiments (git-ignored)

## Quick start

1) Add the repository to your MATLAB path:

```matlab
addpath(genpath(pwd));
```

2) Run all algorithms on a single problem (example):

```matlab
prob.name   = 'afvar1';
prob.dim    = 30;          % dimension
prob.fhandle= @afvar1;     % function handle
prob.startp = zeros(30,1); % starting point

run_all_algorithms_on_problem(prob);
```

This creates `results_placeholder.mat` in `results/` with fields:
`fvec_dse`, `fvec_sds`, `fvec_sto`, `fvec_rnd`, and `fvec_sham`.

3) Batch over a list of problems with multiple runs:

```matlab
run_all_problems_batch(5); % 5 stochastic runs per problem
```

Outputs a combined `.mat` file in `results/`.

Note on StoMADS: you need to place StoMADS core files under `third_party/StoMADS/StoMADS_Main_Files/`.
See `third_party/StoMADS/README_THIRD_PARTY.md` and `HOWTO_APPLY_SAMPLING_RULE.md`.

## Requirements

- MATLAB R2020b or newer recommended
- Statistics and Machine Learning Toolbox (for `sobolset`/`qrandstream`) if using DSE-DD with Sobol directions
- StoMADS third-party files (optional, only if running the StoMADS baseline)

## Profiles and plots

After running a batch (e.g., `run_all_problems_batch(5)`), generate data and performance profiles:

```matlab
make_profiles(5);
```

Figures are written under `results/plots/make_profiles_num_runs_<num_runs>/` as EPS files.

## Noise model

For DSE-DD, SDS, and StoMADS, we simulate a noisy zeroth-order oracle by averaging noisy function values with a sample count that scales with the inverse of step size to the fourth power. Formally, the number of samples is `m(h) = ceil( C^2 * h^{-4} )` with an option for uncorrelated or correlated models.

The helper `sample_computer` returns the exponent used by these algorithms to compute the sample size as `m = ceil(h^{exp})` where the default regime yields `exp = -4`.

The random nonconvex search and Kornowski-Shamir baselines do not use this sample-average oracle in the current harness; they use their own Gaussian noisy-evaluation models.

**Rinaldi, F., Vicente, L. N., & Zeffiro, D. (2023).** "Stochastic trust-region and direct-search methods: A weak tail bound condition and reduced sample sizing."  
arXiv: [2202.11074](https://arxiv.org/abs/2202.11074)

### StoMADS

StoMADS is included as third-party code under `third_party/StoMADS/` and used through a wrapper. Please cite:

arXiv: [1911.01012](https://arxiv.org/abs/1911.01012)

### Kornowski-Shamir
